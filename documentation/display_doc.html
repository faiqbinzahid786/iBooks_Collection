View Collection Documentation


View Collection will display all the contents from the XML file in the form of table. The table wil show all the node tags as table headers and the elements in the table body. Below is the picture of View Collection tab:



The table header represents:

  1. The ID represent the uniqeness of a book. As we have many books, it will be difficult to search for books with the its name. As some books might have same name or event same author.

  2. The Author represent the author or the person who wrote that specific book. Example: Nelson Mandela, William Shakespear, etc.

  3. The Title represent the title of the book that the author wrote. Example: Midnight Rain, A Tale of Two Cities, etc.

  4. The Genre represent the genre of the book. Example: Science, History, Politics, Maths, etc.

  5. The Price represent the price of the books in dollars Example: $10.99, $299.25, etc.

  6. The Published Date represent the date that the book was published on. Example: 12/2/1998, 22/3/2008, etc.

  7. The Description represent the summary or the discription of the book. Example: A former architect battles corporate zombies, an evil sorceress, and her own childhood to become queen of the world.
